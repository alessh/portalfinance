<objective>
Close codex review concern **#12 (LOW — `last_synced_at` as manual-sync cooldown anchor is blunt)**.

Current shipped behavior: `pluggy_items.last_synced_at` is updated on EVERY successful sync (webhook, reconcile, manual, first-connect, reconnect). The manual-sync route at `/api/pluggy/items/:id/sync` checks against `last_synced_at` for the 30-minute cooldown. Two consequences:

1. A failed manual sync attempt does NOT cool down (the worker errored before updating last_synced_at), but a user who clicks "Sincronizar agora" repeatedly while errors persist is not throttled — they consume Pluggy quota for nothing.
2. A successful webhook-driven sync 5 minutes ago BLOCKS the user's manual sync request even though the user has a legitimate need (e.g., they just made a purchase and want to see it). This is a UX defect that the user may attribute to "the app being broken."

Fix:
1. Add nullable column `last_manual_sync_at timestamptz` to `pluggy_items`.
2. pluggy-sync worker writes `last_manual_sync_at = now()` ONLY when `job.data.trigger === 'manual'` AND only inside the success path (after detectors enqueued, before flipping status to UPDATED).
3. Manual-sync route checks `last_manual_sync_at` instead of `last_synced_at` for cooldown.

Behavior choice (Concern #12 said "make the choice deliberate"): we EXPLICITLY accept that failed manual syncs do not cool down — the user retries; if Pluggy is genuinely down they hit their own connector's rate limit, not our cooldown. This is documented in the route's JSDoc.

Output: 1 schema change + drizzle migration + 2 file modifications + cooldown test updates.
</objective>

// distinct from last_synced_at. Failed manual syncs do NOT cool down (the
// worker only writes this column inside its success path); recent
// webhook/reconcile-driven syncs do NOT block subsequent manual sync.

const COOLDOWN_MS = 30 * 60 * 1000;     // D-28 — 30 minutes per item
const last_manual = item.last_manual_sync_at;
if (last_manual) {
  const elapsed_ms = Date.now() - last_manual.getTime();
  if (elapsed_ms < COOLDOWN_MS) {
    const retry_after_seconds = Math.ceil((COOLDOWN_MS - elapsed_ms) / 1000);
    return NextResponse.json(
      { error: 'COOLDOWN_ACTIVE', retry_after_seconds },
      { status: 429, headers: { 'Retry-After': String(retry_after_seconds) } },
    );
  }
}
