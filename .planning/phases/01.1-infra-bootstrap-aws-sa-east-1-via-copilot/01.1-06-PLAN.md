<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| migrate task -> RDS | Task IAM role grants Secrets Manager read + implicit SG ingress; no user-facing exposure |
| ECS Exec session -> running web container | Admin-level shell in prod; scoped by IAM policy on dev's SSO session; session is audited to CloudTrail |
| rerun of migrate -> schema integrity | Drizzle's tracker table guarantees no re-application of already-applied migrations |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-MIGRATE-REPLAY | Tampering | duplicate migration run | mitigate | Drizzle.__migrations ensures no double-apply; Task 3 explicitly tests the rerun produces count=0 |
| T-MIGRATE-PRIV | Elevation of Privilege | migrate task DB user | accept | Task uses the RDS master credentials (portalfinance user) for schema DDL. Phase 6 hardening could split into an app user + migrator user with distinct GRANTs; deferred by scope |
| T-ECS-EXEC-LOG | Information Disclosure | commands run via copilot svc exec | mitigate | CloudTrail records every ECS Exec session; developer SSO session ARN is tied to the trail; Task 2 explicitly avoids querying sample rows (only metadata) |
</threat_model>
