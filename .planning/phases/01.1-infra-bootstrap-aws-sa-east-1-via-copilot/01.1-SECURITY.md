# Phase 01.1 — Security

> Per-phase security contract: threat register, accepted risks, and audit trail.
> Aggregated across 9 plans (00-08); covers AWS Copilot bootstrap to `sa-east-1`, Docker hardening, env contract, RDS, SSM SecureString secrets, IAM task roles, ECR/ECS Exec, Cloudflare TLS, and ops runbooks.

---

## Trust Boundaries

| Boundary | Description | Data Crossing |
|----------|-------------|---------------|
| developer laptop -> AWS account | SSO-based session; short-lived STS credentials only; no long-lived keys | dev session tokens |
| external network -> Next.js /api/health | Public probe; must not leak information nor perform DB queries | static health response |
| build-time -> runtime | Node engine version affects argon2 ABI compatibility (pg-boss 12 requires Node 20+) | binary compat metadata |
| local build pipeline -> production runtime env | env.ts is the runtime gate; refinement loosening could let a misconfigured prod deploy boot | config schema |
| tsup external list -> runtime require() | Bundled deps that should have been externalised can trigger native-binding ABI mismatches | bundled JS modules |
| src/lib/env.ts AWS creds optional -> src/mailer.ts IAM pivot | AWS_ACCESS_KEY_* optional in prod must not allow boot without ANY SES credential path | IAM credentials |
| developer laptop -> Docker daemon | Build-time context contains the repo; .dockerignore gates what enters the image | source files |
| Docker image layers -> public ECR | Layer contents are cached + distributed; any baked secret becomes permanent | image bytes |
| container boot -> AWS Secrets Manager | IAM task role is the only credential; no long-lived keys flow through the container | DB credentials |
| SESClient default provider chain -> IAM task role | No long-lived SES keys in env or SSM; env.ts .optional() pivot delivered by Plan 01.1-02 | IAM credentials |
| manifest YAML -> CloudFormation | Manifest error propagates to a failed `copilot env deploy`; validation catches it pre-deploy | infra config |
| Copilot secret paths -> SSM Parameter Store | `/copilot/portalfinance/prod/secrets/<NAME>` contract; Plan 01.1-05 creates SecureStrings | application secrets |
| environment addon outputs -> task env vars | Misnamed outputs silently break entrypoint.sh -- grep gate catches common misnames | runtime env values |
| developer SSO session -> AWS CloudFormation | Admin-level actions; scoped by STS session TTL | infra mutations |
| local copilot CLI -> AWS APIs | Uses SSO-cached credentials; no long-lived keys touched | infra mutations |
| SSM SecureString write -> KMS | `aws/ssm` managed key by default; KMS access is implicit via IAM | secret material |
| deployed tasks -> RDS | Security group ingress only from the Copilot environment SG | DB credentials, query traffic |
| migrate task -> RDS | Task IAM role grants Secrets Manager read + implicit SG ingress; no user-facing exposure | DDL/DML traffic |
| ECS Exec session -> running web container | Admin-level shell in prod; scoped by IAM policy on dev's SSO session; session is audited to CloudTrail | shell commands |
| rerun of migrate -> schema integrity | Drizzle's tracker table guarantees no re-application of already-applied migrations | DDL ops |
| public internet -> Cloudflare edge | DDoS, WAF, Turnstile at edge; no origin leakage | request traffic |
| Cloudflare edge -> ALB (ACM) | TLS re-encryption; SSL mode MUST be strict to prevent origin-cert spoofing | TLS-encrypted traffic |
| ACM DNS validation record -> Cloudflare DNS | One-time manual CNAME; grey cloud (never proxied) so ACM can validate | DNS metadata |
| ops runbook -> on-call engineer | Misleading instructions -> wrong prod action; scrubbing Railway refs prevents dead-ends | operational guidance |
| encryption-key-rotation.md -> rotation worker / engineer | Incorrect rotation command could leak keys or create a split-brain rotation state | KMS/SSM key material |
