**Alternative (API):** create a scoped API token with `Zone.DNS:Edit` and `Zone.Zone Settings:Edit` on `portalfinance.app`:

```sh
export CF_API_TOKEN="<token>"
ZONE_ID=$(curl -s -H "Authorization: Bearer $CF_API_TOKEN" \
  "https://api.cloudflare.com/client/v4/zones?name=portalfinance.app" | jq -r '.result[0].id')

# Set SSL mode to strict.
curl -sX PATCH "https://api.cloudflare.com/client/v4/zones/$ZONE_ID/settings/ssl" \
  -H "Authorization: Bearer $CF_API_TOKEN" \
  -H "Content-Type: application/json" \
  --data '{"value":"strict"}' | jq '.result.value'
# Expect: "strict"

# Create apex CNAME (proxied).
curl -sX POST "https://api.cloudflare.com/client/v4/zones/$ZONE_ID/dns_records" \
  -H "Authorization: Bearer $CF_API_TOKEN" \
  -H "Content-Type: application/json" \
  --data "{\"type\":\"CNAME\",\"name\":\"@\",\"content\":\"$ALB_DNS\",\"proxied\":true,\"ttl\":1}"
```

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| public internet -> Cloudflare edge | DDoS, WAF, Turnstile at edge; no origin leakage |
| Cloudflare edge -> ALB (ACM) | TLS re-encryption; SSL mode MUST be strict to prevent origin-cert spoofing |
| ACM DNS validation record -> Cloudflare DNS | One-time manual CNAME; grey cloud (never proxied) so ACM can validate |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01.1-CF-TLS | Tampering + Information Disclosure | Cloudflare SSL mode drop to `flexible` | mitigate | Task 2 sets mode to strict; Task 3 grep + API check asserts `"value":"strict"` in validate-phase; dashboard policy documented in runbook |
| T-01.1-DNS-HIJACK | Spoofing | Cloudflare DNS CNAME integrity | mitigate | Cloudflare 2FA enforced at account; CNAME pointing at ALB DNS (which is itself AWS-issued and cert-bound); Origin ACM cert mismatch would trigger 525 (strict mode rejects). |
| T-ACM-VAL-CNAME-PROXY | Denial of Service | DNS validation CNAME accidentally proxied (orange) | mitigate | Task 1 manual step explicitly instructs grey cloud; if proxied, ACM never reaches ISSUED and the plan is blocked (fails closed) |
| T-HEALTH-NO-TLS | Information Disclosure | /api/health reachable via plain HTTP | accept + mitigate | Cloudflare auto-upgrades HTTP to HTTPS when zone has SSL strict; ALB listener on :80 can be dropped in Phase 6 hardening (defer). Acceptance test uses https:// only. |
</threat_model>
